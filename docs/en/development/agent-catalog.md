## Workflow Phases: Typical Deployment Order

### Phase 1: Specification & Review

```mermaid
graph LR
    SPEC["Requirements documents<br/>(spec/)"]

    SPEC --> RCA["requirements-contradiction-analyzer"]
    SPEC --> ARR["agrobiology-requirements-reviewer"]
    SPEC --> ISR["it-security-requirements-reviewer"]
    SPEC --> TSA["tech-stack-architect"]
    SPEC --> TAA["target-audience-analyzer"]
```

The analysis agents run **in parallel** on the specifications and identify gaps, contradictions, and risks.

### Phase 2: Persona Reviews

```mermaid
graph LR
    SPEC["Requirements documents"]

    SPEC --> CIG["cannabis-indoor-grower-reviewer"]
    SPEC --> CHU["casual-houseplant-user-reviewer"]
    SPEC --> OGP["outdoor-garden-planner-reviewer"]
    SPEC --> SHA["smart-home-ha-reviewer"]
    SPEC --> FDR["frontend-design-reviewer"]
```

Persona reviews run **in parallel** and check whether all target audiences are covered. Results feed back as spec extensions.

### Phase 3: HA Integration

```mermaid
graph LR
    SHA["smart-home-ha-reviewer<br/><i>Review</i>"]
    SHA --> HIRE["ha-integration-requirements-engineer<br/><i>Derive requirements</i>"]
```

The HA requirements engineer builds on the review from the smart-home reviewer.

### Phase 4: Implementation

```mermaid
graph LR
    SPEC["Specifications + Reviews"]
    SPEC --> FSD["fullstack-developer<br/><i>Feature implementation</i>"]
    FSD --> FUO["frontend-usability-optimizer<br/><i>UX optimization</i>"]
    FSD --> CSR["code-security-reviewer<br/><i>Security audit</i>"]
```

The fullstack developer implements; afterwards the usability optimizer refines the UI and the code security reviewer audits security.

### Phase 5: Testing

```mermaid
graph LR
    SPEC["Specifications"]
    SPEC --> TCE["e2e-testcase-extractor<br/><i>Test case derivation</i>"]
    TCE --> STG["selenium-test-generator<br/><i>Test implementation</i>"]
    STG --> STR["selenium-test-reviewer<br/><i>QA validation</i>"]
```

Test cases are derived from specs, implemented as Selenium tests, and then checked for compliance.

### Phase 6: Documentation & Assets

```mermaid
graph LR
```
