## F3: Fragen-Logging (MITTEL, Aufwand M)

### Problem
Nutzerfragen werden im Klartext geloggt. Keine Retention-Policy.

### Aenderungen

- `src/knowledge-service/app/config.py`: `log_questions_plaintext: bool = False`
- `src/knowledge-service/app/service.py`: Helper-Funktion `_redact_question()`:
  ```python
  def _redact_question(question: str, plaintext: bool) -> str:
      if plaintext:
          return question
      return f"sha256:{hashlib.sha256(question.encode()).hexdigest()[:16]}... (len={len(question)})"
  ```
- Betroffene Log-Zeilen in service.py: 79, 89, 105-111, 117, 141, 166-175
- NFR-011 Retention-Matrix: Neue Kategorie "R-23: RAG-Anfragen" mit 30 Tagen Aufbewahrung

### Tests
- Unit-Test: Bei `log_questions_plaintext=False` kein Klartext im Log-Event
- Unit-Test: Bei `log_questions_plaintext=True` Klartext vorhanden

## F5: LLM-Fallback (NIEDRIG, Aufwand S)

### Problem
Bei LLM-Timeout 500er statt hilfreicher Fallback.

### Aenderungen

- `src/knowledge-service/app/service.py`: try/except um `self._llm.generate()`:
  ```python
  try:
      response = self._llm.generate(...)
  except (httpx.TimeoutException, httpx.HTTPStatusError):
      return KnowledgeAnswer(
          answer=fallback_message,
          sources=chunks,  # Chunks trotzdem zurueckgeben!
          model="fallback",
          is_fallback=True,
      )
  ```
- `src/knowledge-service/app/prompt_engine.py`: `get_fallback_message(language)`
  - DE: "Der KI-Assistent ist derzeit nicht erreichbar. Die gefundenen Quellen koennen Ihnen moeglicherweise bereits weiterhelfen."
- `src/knowledge-service/app/schemas.py`: `is_fallback: bool = False` in `AskResponse`

### Tests
- Unit-Test: LLM wirft TimeoutException -> `is_fallback=True`, Sources vorhanden
- Unit-Test: Normaler Aufruf -> `is_fallback=False`
