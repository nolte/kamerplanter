## NFR-008 is a Meta-NFR — Tester Perspective is QA-Engineer at Terminal

NFR-008 defines the test infrastructure itself. Test cases do NOT describe browser interactions but rather:
- What a QA-Engineer sees when running `pytest` commands in the terminal
- What appears on the filesystem (protokoll.md, screenshots/, test-reports/)
- Whether Page-Object-Pattern is correctly enforced (grep for direct Selenium calls)
- Whether coverage thresholds are met

**Why:** This requires a perspective shift from "user in browser" to "developer/QA at terminal" — the SuT is the test infrastructure, not a UI feature.

**How to apply:** When extracting tests from meta-NFRs (test strategy, CI, deployment), use the QA-Engineer/developer perspective. Only Gruppe 8 (E2E Kernfunktionen) test cases actually involve a browser.

## Key File Locations (NFR-008 Implementation Targets)
- `tests/e2e/conftest.py` — browser fixture, screenshot fixture, failure hook, pytest_addoption
- `tests/e2e/pages/base_page.py` — BasePage with navigate(), wait_for_element(), take_screenshot()
- `tests/e2e/pages/dashboard_page.py` — DashboardPage locators: `[data-testid='plant-count']`, `[data-testid='location-list']`
- `tests/e2e/pages/plant_detail_page.py` — PlantDetailPage locators: `[data-testid='transition-button']`, `[data-testid='confirm-button']`, `[data-testid='current-phase']`
- `tests/e2e/protocol_plugin.py` — ProtocolGenerator, pytest_configure, pytest_sessionstart, pytest_sessionfinish
- `tests/factories.py` — BotanicalFamilyFactory, PlantFactory, LocationFactory (counter-based, DB-free)
- `tests/conftest.py` — sample_family, sample_plant, sample_location fixtures
- `tests/integration/conftest.py` — clean_collections autouse fixture (Collection-Truncate)
- `test-reports/` — must be in .gitignore; timestamped subfolders YYYY-MM-DD_HH-MM-SS

## Screenshot Naming Convention (NFR-008 §3.4)
- Page-Load: `001_dashboard-overview.png`
- Before action: `002_plant-detail-before-transition.png`
- Dialog: `003_transition-confirm-dialog.png`
- After action: `004_plant-detail-after-transition.png`
- Failure: `FAILURE_{test_node_name}.png` (auto-generated by hook)

## CLI Options for E2E Suite (NFR-008 §3.1, §6.3)
- `--browser`: chrome (default, headless=new) | firefox (headless)
- `--base-url`: default `http://localhost:5173`
- `--generate-protocol`: activates ProtocolGenerator; without this flag no protokoll.md is created
